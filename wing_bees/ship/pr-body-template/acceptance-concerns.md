### Placeholder definitions

- **`{{summary}}`** — drawn from the plan's `## Context` first paragraph. Describes what the feature does and why it matters. 1–2 paragraphs.

- **`{{specs}}`** — a bullet list of specs that shipped. Format: `- [spec NNN](.bees/<slug>/specs/NNN-kebab-name.md) — one-line summary`. Links go to spec files in the repo.

- **`{{test_plan}}`** — consolidated test plan from each spec's `## Failing test` section. Describes verification approach so reviewers understand what was tested.

- **`{{linked_issues}}`** — extracted from the briefing. Issues that are closed by this PR (`Closes #N`) appear here with auto-close syntax preserved.

- **`{{risks}}`** — drawn from the plan's risk section if present. Describes tradeoffs, dependencies, or areas where the change may have subtle effects.

- **`{{acceptance_concerns}}`** — present only when queen-accept emits CONCERNS; carries the numbered list verbatim from `.bees/<slug>/accept.md`. Absent if queen-accept approved.

- **`{{version}}`** — bees version that generated the PR (e.g. `0.1.0`).

- **`{{plan_link}}`** — relative link to the plan file: `.bees/<slug>/plan.md`. Lets reviewers read the full planning context without leaving GitHub.

- **`{{trace_link}}`** — relative link to the honeycomb recall trace: `.bees/<slug>/honeycomb-trace.md`. Shows what S-SDLC rooms shaped the plan.

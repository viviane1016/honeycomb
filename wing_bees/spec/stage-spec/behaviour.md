## Behaviour

`_extract_work_units` reads the plan and extracts the N work-breakdown items. N scribes run via ThreadPoolExecutor with parallelism tunable via `BEES_SPEC_PARALLEL` (default 4) and `--parallel N`. Each scribe runs as a fresh `claude -p` subprocess; the model is resolved per work-breakdown item via `cli_override > [scribe:<m>] tag > default (claude-opus-4-7)`. The compound tag form `[scribe:<scribe-model>, <builder-model>]` sets both tiers independently; the bare form `[<builder-model>]` keeps the scribe at Opus by default. Each scribe receives the plan plus one item and emits exactly one fenced `<<<SPEC NNN-…>>>` block with seven required sections: Builder model, Goal, Scope, Failing test, Builder prompt, Success check, Commit message.

Per-scribe parse failures retry once with error feedback. Draft specs are held in memory by the bees harness. The queen then runs a read-only review pass on all drafts and emits per-NNN APPROVE or REWRITE blocks. REWRITE bodies are re-validated and secret-scanned; coverage mismatches (missing, surplus, duplicate NNN) or validation failures abort the spec stage with no files written. On success, all spec files are written atomically. Secret scan applies at output (each scribe draft, before disk write) and boundary (each queen REWRITE body, before apply) gates; see `wing_practices/secret-handling`.

**Depends-on inference pass.** Before queen-review, `_infer_depends_on_from_output_files` computes the `output-files` overlap graph across all drafts and injects programmatic DEPS patches for unambiguous overlaps (larger NNN depends on smaller NNN by numeric order). Drafts missing `output-files` declarations that share files with other specs cause spec-stage to hard-fail with a clear gap-pointing error. A log line is emitted per inferred patch so the operator can audit the inference in the spec-stage transcript. See `queen-review` for the full algorithm.

**Coverage-gap halt.** Before the queen review fires, `_verify_spec_coverage(plan_md, drafts)` extracts every plan NNN and checks each appears as a draft NNN. On any drop, it emits a `spec` event with `outcome="coverage_gap"` and exits non-zero with a `bees spec: scribe drops detected for NNN [, NNN…]; rerun with bees spec --force <slug>` message.

A `stage="spec_review"` event is emitted with `outcome ∈ {started, success, error, timeout, parse_failure_final, secret_detected, rewrite_invalid}` and (on success) a `rewrite_count` field. The spec stage also emits per-scribe records carrying `scribe=NNN` plus an aggregate `outcome=success` record summing usage across all scribes — review tokens are excluded from the aggregate and appear only on the `spec_review` record.

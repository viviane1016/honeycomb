## Default conventions practice

Honeycomb v0.1 ships with defaults for: (1) branch naming (`feat/auth-oauth`, `fix/dashboard-timeout`, not `john-branch` or `wip`), (2) commit type vocabulary (feat/fix/refactor/docs/test/chore/build/ci/perf/style/revert), (3) merge strategy (rebase-and-merge preserves history; merge commits create a merge bubble; squash is discouraged because it loses the individual commit messages). When a repo has established a different convention (e.g. it's squashed everything for five years, or it uses a ticket-prefix convention), the queen-file's "Established patterns" section records that exception. The scout stage (future) will detect intra-repo inconsistency and surface conflicts to the operator; the queen and operator agree on the override, and it's recorded in the queen-file so future bees invocations respect it.

Defaults: branch naming `feat/<slug>`, `fix/<slug>`; commit messages follow Conventional Commits; merge by rebase-and-merge or merge commit (not squash, which loses history). Scopes come from the briefing's classifier tag or the feature slug. The queen-file (`QUEEN.md` in repo root) is the escape hatch—Established patterns section records repo-specific overrides, audit exemptions, or deviations from honeycomb defaults.

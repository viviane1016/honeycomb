Every public-behaviour change ships with a docs-update spec. Public means: CLI surface (new flags, subcommands, --help text), workflow steps the user runs by hand, stage behaviour affecting human gates, builder tool allowlist, or claims in README.md / CLAUDE.md / SKILL.md. The docs-update spec touches the files the change affects and marks shipped items in BACKLOG.md. Pure internal refactors and bugfixes are exempt.
